# SPEC.md
Peekdiff (macOS / Tauri + React + TypeScript + Tailwind)

---

# 1. プロジェクト概要

## 1.1 目的

macOS向けに、WinMergeのようなシンプルで直感的な「ディレクトリ比較ツール」を提供する。

本アプリは以下を重視する：

- 学習コストが低い
- 差分が一目で分かる
- 大規模フォルダでもフリーズしない

## 1.2 技術スタック

| レイヤー | 技術 |
|---------|------|
| デスクトップシェル | Tauri (Rust) |
| UI フレームワーク | React 18 + TypeScript |
| ビルドツール | Vite |
| スタイリング | Tailwind CSS |
| UIコンポーネント | shadcn/ui |
| アニメーション | framer-motion (motion/react) |
| アイコン | lucide-react |
| パッケージ管理 | pnpm |

# 2. 設計原則

## 2.1 シンプル第一

- UIは明確な階層構造
- 機能はMVPから段階的に拡張
- 過剰設計をしない

## 2.2 責務分離

| 層 | 役割 |
|----|------|
| UI層（React） | 表示・操作 |
| IPC層（Tauri invoke） | フロントエンド ↔ バックエンド通信 |
| 比較エンジン（Rust/Tauri） | ファイルシステム走査・差分計算 |
| 状態管理（React useState） | フィルタ・選択状態 |

## 2.3 モックファースト

- Figma Make で定義したモックデータ（mockData.ts）でUIを先行開発する
- Tauri IPC は後続 issue で差し替える

# 4. MVP機能要件

## 4.1 比較モード選択

ヘッダーにタブを設け、2つの比較モードを切り替える：

| モード | 説明 |
|--------|------|
| フォルダ比較 | 左右のディレクトリを再帰的に比較（デフォルト） |
| ファイル比較 | 左右の単一ファイルを直接比較 |

モード切り替え時は入力パス・比較結果・エラーをすべてリセットする。

## 4.2 パス入力

- 左右2ペインでパス（フォルダまたはファイル）を入力
- Tauri IPC 経由でOSネイティブダイアログ（フォルダ選択 or ファイル選択）を呼び出す
- Finder からウィンドウにドラッグ＆ドロップしてパスを入力できる（ドロップ位置の左右で自動振り分け）

## 4.3 ディレクトリ比較

比較ステータス分類：

| ステータス | 意味 | 色 |
|-----------|------|----|
| `added` | Bにのみ存在 | 緑 |
| `deleted` | Aにのみ存在 | 赤 |
| `modified` | 両方存在するが内容が異なる | 黄 |
| `identical` | 完全一致 | グレー |

## 4.4 差分ツリー表示（DirectoryTree）

- ツリー形式でファイル・ディレクトリを表示
- ステータス別カラー表示（左ボーダー + 背景グラデーション）
- ディレクトリはクリックで開閉
- 選択ファイルはハイライト表示

## 4.5 詳細ビュー（FileDetailView）

### テキストファイル（TextDiffView）

- 左右2カラムのSplit diff表示
- 行番号表示
- 追加行：緑背景、削除行：赤背景、変更なし：透明

### バイナリファイル（BinaryFileView）

- ファイルサイズ（KB）
- ハッシュ値（SHA-256）
- 最終更新日時

## 4.6 ファイル直接比較

- ファイル比較モードで左右のファイルパスを入力し「比較」を実行する
- 比較結果は `FileDetailView` に全幅で表示する（DirectoryTree は非表示）
- テキストファイルは Split Diff、バイナリは情報表示（既存 `FileDetailView` をそのまま利用）
- フィルタ・検索・統計バッジはファイル比較モードでは非表示

## 4.7 フィルタ機能

- すべて / 追加 / 削除 / 変更 / 同一 をセレクトで切り替え

## 4.8 検索

- ファイル名部分一致でツリーをリアルタイムフィルタ

## 4.9 比較実行

- 「比較」ボタンで再スキャン
- スキャン中はボタン無効化 + プログレスバー表示
- 比較完了時にConfettiアニメーションを表示
- 完了後に統計バッジを更新（フォルダ比較モードのみ）

## 4.10 統計バッジ

ヘッダーに常時表示：

- `+N`（追加数）
- `-N`（削除数）
- `~N`（変更数）
- `=N`（同一数）

- カラーテーマ：パープル〜ピンク〜ブルーのグラデーション

# 6. 非機能要件

## 6.1 パフォーマンス

- 比較はTauriバックエンド（Rust）で実行し、UIをブロックしない
- 段階比較アルゴリズム（ディレクトリ比較）：
  1. パス存在比較
  2. サイズ比較
  3. ハッシュ比較（サイズが一致した場合）
  4. テキスト判定後diff生成
- テキストdiffは5行先読みアルゴリズムでコンテキストを保持
- ハッシュ：SHA-256、大容量ファイルはストリーム処理
- テキストファイルの読み込み上限：1 MB（超過時はメッセージ表示）

## 6.2 Tauri IPC コマンド一覧

| コマンド | 説明 |
|---------|------|
| `compare_directories` | 左右ディレクトリを再帰比較し `FileNode[]` を返す |
| `compare_files` | 左右ファイルを直接比較し `FileNode` を返す |
| `open_folder_dialog` | OSネイティブのフォルダ選択ダイアログを開く |
| `open_file_dialog` | OSネイティブのファイル選択ダイアログを開く |
| `read_file_content` | 指定パスのテキストファイルを読み込む |

## 6.3 macOS最適化

- Retina対応
- Tauri標準ウィンドウ、最小サイズ 640×500
- Finder D&Dはウィンドウの左半分 / 右半分でパス振り分け（Tauri `onDragDropEvent` 使用）

## 6.4 安定性

- エラー時はクラッシュしない
- アクセス不可ファイルはスキップしてログ表示
- `.git` / `node_modules` / `target` 等は自動スキップ
