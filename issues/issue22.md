# issue22: ウィンドウリサイズのスナップバック修正 & Dock アイコン縮小

## 問題 1: ウィンドウのスナップバック

### 原因

PR #23 で追加した `minWidth: 800` が大きすぎる。
ユーザーが 800px 以下に縮めようとすると OS が強制的に 800px に戻すため、
「元のサイズに戻る」と感じる。

CSS レイアウトの実際の最小幅：
- 左ペイン（w-96）= 384px + gap(4px) + 右ペイン最低(~200px) + padding(8px) ≈ **596px**

→ `minWidth: 640` に変更（CSS最小幅より少し大きい値で自然な制限）

### 修正

```json
"minWidth": 640,
"minHeight": 500
```

## 問題 2: Dock アイコンの絵柄が大きい

### 原因

Moon が 256×256（512px キャンバスの 50%）で大きく見える。

### 修正

| 要素 | 変更前 | 変更後 |
|------|--------|--------|
| Moon | 256×256 at (128,128) | 200×200 at (156,156) |
| Star | 140×140 at (321,51) | 110×110 at (351,51) |

約 80% に縮小してキャンバス内の余白を増やす。

## 変更対象ファイル

- `src-tauri/tauri.conf.json`
- `src-tauri/icons/AppIcon.svg`
